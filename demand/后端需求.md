# 角色与背景
你现在是一位资深的 Java 后端架构师。我正在开发一个基于大模型的“AI 宠物医生”微信小程序。
我的技术栈是：JDK 17 + Spring Boot 3.5 + WebFlux (响应式编程)。
目前我已经本地部署了 Dify，需要你帮我编写一个核心的【多模态 AI 问诊流式接口】。

# 业务场景（请参考图文交互逻辑）
用户在前端选中了宠物（例如：小贝，金毛），输入了病情描述（文本），并附带了一张宠物患处的照片（图片）。
前端会调用我的 Spring Boot 接口，我需要将这些信息整合后，请求 Dify 的 API，并将 Dify 的回复以 SSE (Server-Sent Events) 的形式流式返回给前端。

# 需求细节与接口规范
请为我编写对应的 Controller、Service 和必要的 DTO 类，满足以下要求：

1. 接口定义：
    - 路径：`POST /api/v1/chat/completions`
    - 接收参数 (JSON)：
        - `userId` (String): 用户 ID
        - `petId` (Long): 宠物 ID
        - `query` (String): 用户输入的病情描述
        - `imageUrl` (String): 用户上传的图片地址（可选参数）

2. 上下文组装逻辑 (Service 层)：
    - 请在 Service 中写一个 Mock 方法 `getPetProfile(Long petId)`，模拟从数据库获取宠物信息（返回字符串，例如："宠物名：小贝，品种：金毛，年龄：3岁"）。
    - 需要将获取到的宠物信息，拼接到 Dify 请求体中的 `inputs` 变量中（对应 Dify 的变量输入）。

3. 对接 Dify API 要求：
    - 使用 Spring WebFlux 的 `WebClient` 发起对 Dify 的非阻塞请求。
    - Dify 接口地址：`POST https://api.dify.ai/v1/chat-messages` (请将 baseUrl 提取到配置文件或常量中)
    - Header 需携带：`Authorization: Bearer {dify_api_key}`
    - Dify 请求体 JSON 结构如下（请严格参照）：
      {
      "inputs": {
      "pet_profile": "这里填入查到的宠物档案字符串"
      },
      "query": "用户的病情描述",
      "response_mode": "streaming",
      "user": "{userId}",
      "files": [
      // 如果前端传了 imageUrl，则拼装这块；如果没有，则传空数组或省略
      {
      "type": "image",
      "transfer_method": "remote_url",
      "url": "{imageUrl}"
      }
      ]
      }

4. SSE 流式响应处理：
    - 将 Dify 返回的流（`text/event-stream`）逐块读取，并实时以 `Flux<ServerSentEvent<String>>` 的形式返回给我的前端。
    - 注意异常处理机制，如果 Dify 请求超时或报错，需向前端下发一个标准的错误流信息。
